# Wiring

## Pin Allocation Strategy

The project uses separate pins for each interface to avoid conflicts between team members.

General rules:
- Do not use STM32 SWD debug pins: PA13 / PA14.
- Keep PWM pins reserved for the PWM task.
- Keep SPI pins reserved for the SPI task.
- Use separate ESP32 pins for test control, DIO, ADC/DAC, UART, and SPI.
- All boards must share a common GND.

---

# DIO Button-to-LED Test

## Purpose

The ESP32 testbench simulates a button input to the STM32 DUT.
The STM32 DUT reads this input and drives an LED output.
The ESP32 reads the DUT output and validates the result.

## Signal Mapping

| Signal Name | Source | Destination | Description |
|---|---|---|---|
| BUTTON_SIM | ESP32 GPIO27 | STM32 PB0 | Simulated button signal generated by the testbench |
| LED_OUT | STM32 PB1 | ESP32 GPIO14 | DUT output monitored by the testbench |
| GND | ESP32 GND | STM32 GND | Common ground |

## Saleae Connections

| Saleae Channel | Signal |
|---|---|
| CH0 | BUTTON_SIM |
| CH1 | LED_OUT |

## Expected Behavior

| BUTTON_SIM / STM32 BUTTON_IN | STM32 LED_OUT | Expected Result |
|---|---|---|
| LOW | LOW | PASS |
| HIGH | HIGH | PASS |

# ADC Sensor Threshold Test

## Purpose

The ESP32 testbench generates an analog voltage using its DAC.
The STM32 DUT reads this signal using ADC.
Based on a threshold, the STM32 drives a digital warning output.
The ESP32 monitors this warning output and validates the result.

## Signal Mapping

| Signal Name | Source | Destination | Description |
|---|---|---|---|
| SENSOR_SIM | ESP32 GPIO25 / DAC1 | STM32 PA0 / ADC_IN0 | Simulated analog sensor signal |
| WARNING_OUT | STM32 PA1 | ESP32 GPIO34 | DUT warning output monitored by the testbench |
| GND | ESP32 GND | STM32 GND | Common ground |

## Saleae Connections

| Saleae Channel | Signal |
|---|---|
| Analog CH0 | SENSOR_SIM |
| Digital CH1 | WARNING_OUT |

# UART Ping-Pong Test

## Purpose

The ESP32 testbench sends UART data to the STM32 DUT.
The STM32 replies with the expected response.
The ESP32 validates the received response.

## Signal Mapping

| Signal Name | Source | Destination | Description |
|---|---|---|---|
| UART_TX_ESP32 | ESP32 GPIO17 | STM32 PA3 / USART2_RX | Testbench UART TX |
| UART_TX_STM32 | STM32 PA2 / USART2_TX | ESP32 GPIO16 | DUT UART TX |
| GND | ESP32 GND | STM32 GND | Common ground |

## Saleae Connections

| Saleae Channel | Signal |
|---|---|
| CH0 | ESP32 UART TX |
| CH1 | STM32 UART TX |
